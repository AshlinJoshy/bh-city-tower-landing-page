# Replacing the placeholder images

Every image on this page is a placeholder right now — a gradient SVG
generated by `scripts/make-placeholders.mjs`. They live in
`public/images/` and use warm tones that loosely match the Figma
palette so the page looks intentional even before real photos arrive.

The components use a small wrapper (`src/components/Img.tsx`) that
always tries to load **`<name>.png`** first, and falls back to the
placeholder **`<name>.svg`** if the PNG is missing. That means you can
swap photos in one at a time — the page never breaks.

## How to swap in real Figma photos (no coding needed)

1. **Open the Figma file** → page `Final` → frame
   `CIty Tower landing page op3-1`.
2. **Click a layer** in the canvas (e.g. one of the dining-area photos).
3. **Right sidebar → Export** → set format to `PNG` → click
   `Export <layer name>`.
4. **Rename** the saved file so it matches a name in the table below
   (e.g. `hero-tower.png`).
5. **Upload** it into the `public/images/` folder. You can do this
   straight from GitHub.com:
   - go to the repo → `public/images/` folder
   - click **Add file → Upload files**
   - drag your renamed `.png` into the box
   - **Commit changes** at the bottom
6. Vercel will auto-redeploy and the real photo replaces the
   placeholder. No code changes required.
